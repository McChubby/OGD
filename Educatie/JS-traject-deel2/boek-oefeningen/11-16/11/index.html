Video formats

  • Ogg/Theora(.ogv extension), free without patent issues. this format is supported by Firefox, Chrome and Opera browsers.
  • WebM/VP8(.webm extension), WebM is royalty-free under Berkeley Software Distribution(BSD) open-source license. Works well on low-power devices, including netbooks and handhelds. Works on Firefox, Chrome, Opera and Android an IE 9+
  • MPEG-4/H.264(.mp4 extension), capable of providing good video quality at substanitally lower bit rates than previous stands. has patented technologies, H.264-encoded internet video that is free to end users will never be charged royalties. Supported by IE, Chrome and Safari, Chrome has announced its intent to remove support for the extension in the near future.
Summary:
MPEG-4/H.264 is the most common format, it has the better perfomance(data-stream size to picture quality) although it isn't a big difference. many existing mobie devices have MPEG-4/H.264 hardware decoders, which cannot be ignored.

Implementing the <video> element

simple implementation of the video element


            <video width="320" height="240" controls="controls">
              <source src="movie.mp4" />
              You need a browser that supports HTML5!
            </video>

Result

Configuring the <video> element

  • autoplay, start the video immediately.
  • controls, display play/pause button, video cursor and fullscreen
  • height, set height
  • loop, video will repeat
  • muted, audio in video is silenced
  • poster,specifies to show URL of an image is to be shown when the video is not playing
  • preload, how the video should load auto, metadata or none
  • src, specifies the URL of the video.
  • width, indicates the width in pixels of the rendered <video> element.

Setting the source

the element specifies a video source, you should provide more then one source this way the browser can shoose the format it can use. The recommended order is:

  • .webm(royalty-free),
  • .ogv(royalty-free),
  • .mp4(not always royalty-free).

Accessing tracks

W3C developed a new standard called WebVTT(Web Video Text Tracks). all browsers implementing <track> element, will support it. WebVTT is based on a popular subtitling format called SubRip Text(SRT).

Using the WebVTT format.

              WEBVTT File

              00:00:07.500 --> 00:00:08.750
              He's fidgety.

              NOTE I think the timing should be adjusted to span to the end of the video
              00:00:09.000 --> 00:00:12.000
              There he goes!
The WebVTT format is evolving and provides for other features that are not currently supported on most browsers.

Using the HTMLMediaElement object

The <video> & <audio> elements inherit from the HTMLMediaElement object.

Understanding the HTMLMediaElement methods

  • addTextTrack(), Adds a new text track to the audio or video
  • canPlayType(), Determines wheter the browser can play the specified audio or video type
  • load(), Reloads the audio or video
  • play(), Plays the audio or video
  • pause(), Pauses the currentyle playing audio or video
Using HTMLMediaElement properties

In addition to the methods that enable you to control playback, you can us many properties to view or set the state of the <audio> or <video> element.

Subscribing to HTMLMediaElement events

In addition to the methods properties of HTMLMediaElement, it is also possible to subscribe to events.
